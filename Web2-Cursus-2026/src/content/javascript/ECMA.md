ECMA staat oorspronkelijk voor European Computer Manufacturers Association, een organisatie die standaarden voor computers en communicatie ontwikkelde; hoewel de organisatie nu bekendstaat als Ecma International om zijn wereldwijde focus te weerspiegelen, is de historische naam behouden voor de vele invloedrijke standaarden.

## Wat is ECMA-Script?

ECMA-Script is de standaard waarop JavaScript is gebaseerd.  
De standaard wordt beheerd door de ECMA International organisatie.  
De eerste versie van de standaard werd in 1997 gepubliceerd en sindsdien zijn er verschillende versies uitgebracht.

ECMA-Script is belangrijk omdat het ervoor zorgt dat JavaScript consistent is in verschillende browsers.  
Dit betekent dat ontwikkelaars er zeker van kunnen zijn dat hun code op dezelfde manier zal werken in verschillende browsers.

De nieuwste versie van ECMA-Script is **ES14**, die in 2023 werd uitgebracht.

Toch focussen we in deze cursus voornamelijk op **ES6**, de versie die in 2015 werd uitgebracht en die ondersteund wordt door de meeste moderne browsers.

## Client side en Server side

JavaScript kan zowel aan de client-side als aan de server-side worden uitgevoerd.

- **Client-side JavaScript** wordt uitgevoerd in de browser van de gebruiker en wordt gebruikt om interactieve webpagina's te maken.
- **Server-side JavaScript** wordt uitgevoerd op de server en wordt gebruikt om de server te beheren en gegevens te verwerken ("back-end code").

Node.js is een populaire runtime-omgeving die het mogelijk maakt om JavaScript aan de server-side uit te voeren.  
Dit wordt uitvoerig besproken in de cursus Web 3.
